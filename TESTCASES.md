## Тест-кейсы 
### 1. Создание объявления

#### 1.1. Успешное создание объявления
**Предусловие:** Все обязательные поля заполнены корректными данными
**Шаги:**
1. Отправить POST запрос на /api/1/item
2. В теле запроса указать:
   - sellerID: уникальный числовой ID (111111-999999)
   - name: строка с названием
   - price: положительное число
   - statistics: объект с likes, viewCount, contacts

**Ожидаемый результат:**
- Status code: 200
- В ответе присутствуют все переданные поля
- Присутствует поле id (строка)
- Присутствует поле createdAt (строка с датой)

#### 1.2. Создание объявления с невалидным sellerID
**Предусловие:** sellerID содержит нечисловое значение
**Шаги:**
1. Отправить POST запрос на /api/1/item
2. В теле запроса указать невалидный sellerID

**Ожидаемый результат:**
- Status code: 400

#### 1.3. Создание объявления без обязательных полей
**Предусловие:** Отсутствует одно из обязательных полей
**Шаги:**
1. Отправить POST запрос на /api/1/item
2. В теле запроса пропустить одно из обязательных полей

**Ожидаемый результат:**
- Status code: 400

### 2. Получение объявления по ID

#### 2.1. Успешное получение существующего объявления
**Предусловие:** Создано объявление с известным ID
**Шаги:**
1. Отправить GET запрос на /api/1/item/:id
2. Передать валидный ID существующего объявления

**Ожидаемый результат:**
- Status code: 200
- В ответе присутствуют все поля объявления
- Значения полей соответствуют созданному объявлению

#### 2.2. Получение несуществующего объявления
**Предусловие:** ID не существует в системе
**Шаги:**
1. Отправить GET запрос на /api/1/item/:id
2. Передать несуществующий ID

**Ожидаемый результат:**
- Status code: 404

#### 2.3. Получение объявления с невалидным ID
**Предусловие:** ID имеет неверный формат
**Шаги:**
1. Отправить GET запрос на /api/1/item/:id
2. Передать невалидный ID (спецсимволы, пустая строка)

**Ожидаемый результат:**
- Status code: 400

### 3. Получение всех объявлений продавца

#### 3.1. Успешное получение объявлений существующего продавца
**Предусловие:** Существует продавец с несколькими объявлениями
**Шаги:**
1. Отправить GET запрос на /api/1/:sellerID/item
2. Передать валидный sellerID

**Ожидаемый результат:**
- Status code: 200
- Ответ содержит массив объявлений
- Все объявления принадлежат указанному sellerID

#### 3.2. Получение объявлений несуществующего продавца
**Предусловие:** sellerID не существует
**Шаги:**
1. Отправить GET запрос на /api/1/:sellerID/item
2. Передать несуществующий sellerID

**Ожидаемый результат:**
- Status code: 200
- Ответ содержит пустой массив

#### 3.3. Получение объявлений с невалидным sellerID
**Предусловие:** sellerID имеет неверный формат
**Шаги:**
1. Отправить GET запрос на /api/1/:sellerID/item
2. Передать невалидный sellerID

**Ожидаемый результат:**
- Status code: 400

### 4. Получение статистики по объявлению

#### 4.1. Успешное получение статистики существующего объявления
**Предусловие:** Создано объявление с известным ID
**Шаги:**
1. Отправить GET запрос на /api/1/statistic/:id
2. Передать валидный ID существующего объявления

**Ожидаемый результат:**
- Status code: 200
- Ответ содержит массив с объектами статистики
- Каждый объект содержит поля likes, viewCount, contacts

#### 4.2. Получение статистики несуществующего объявления

1. Отправить GET запрос на /api/1/statistic/:id
2. Передать несуществующий ID

**Ожидаемый результат:**
- Status code: 404

### 5. Удаление объявления

#### 5.1. Успешное удаление существующего объявления

**Шаги:**

- Status code: 200

- Status code: 404

### 6. Интеграционные тесты

#### 6.1. Полный цикл создания и получения объявления
**Шаги:**
1. Создать объявление
2. Получить созданное объявление по ID
3. Проверить соответствие данных

**Ожидаемый результат:**
- Все шаги выполняются успешно
- Данные в полученном объявлении соответствуют созданным

#### 6.2. Создание нескольких объявлений одного продавца
**Шаги:**
1. Создать 2 объявления с одинаковым sellerID
2. Получить все объявления продавца
3. Проверить количество и содержание объявлений

**Ожидаемый результат:**
- В ответе содержится 2 объявления
- Оба объявления принадлежат указанному sellerID
